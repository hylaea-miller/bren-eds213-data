For the variable $1 it shows the first CSV filename (e.g. a.csv). 
For the variable $# it shows the total number of CSV files in the directory (e.g. 5).

Question 7.
Answer:
I got an empty file, it happened because the > signal can be destructive. In this case, bash overwrites the existing file with an empty file before the program is run
You can fix it by overwriting in a temporary file and then replacing it with mv
e.g. 
sort junk_file.txt > tmpfile
mv tmpfile junk_file.txt
